Optional. The table/column/filter are built into a self-contained condition (SELECT <expr> FROM table WHERE filter)>N, so your template stays generic (e.g. FROM dual) — you do not repeat the table/WHERE in the template. With a filter the tool targets that one row; without it, it pages through rows.

Each line: column || FROM …. The tool tries them in order and keeps the first that passes validation (LENGTH>0 true and LENGTH>9999 false). Leave blank to use built-in defaults for the selected DB. Example (Oracle):
table_name || FROM (SELECT table_name, ROWNUM rn FROM user_tables) WHERE rn={rownum}
